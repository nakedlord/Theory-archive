## Core Claim

R Theory is strongest when R levels are read not as abstract intelligence
levels, but as modes for moving learning away from direct real-world error.

```text
R0: reaction, no meaningful choice window
R1: action/result learning and stabilization
R2: internal simulation of action sequences and local futures
R3: analysis and rewrite of the rules that generate behavior
```

The evolutionary advantage is not "more thought" in the vague sense. It is a
reduction in the number and cost of real mistakes required to improve behavior.

## R3 Mode Split

The export makes the R3 split explicit enough to preserve as a candidate.

### R3.1: Online Control

Function:

```text
prediction(t) <-> observation(t)
```

R3.1 keeps attention in the present and checks whether the near-future model
generated by R2 is matching reality. When prediction and observation match, the
world feels causally stable.

Candidate implication:

```text
felt reality = ongoing prediction adequacy
```

### R3.2: Local Error Analysis

Trigger:

```text
observation - prediction exceeds threshold
```

R3.2 fixes the mismatch, asks why the model failed, and rewrites a local rule.
It is not reflection for its own sake. It is second-order learning after model
failure.

### R3.3: Offline Retrospective

Trigger:

- long absence of salient mismatch;
- unresolved critical error;
- quieted present stream;
- background pressure from previous model failures.

R3.3 reopens past errors and can become deep retrospective analysis or extended
rumination. Each recursive step fixes a state and launches a new choice from
that state, which explains the felt endlessness of some thinking loops.

## Motivational Field

The R3.1-3 source adds an important correction:

```text
R2 does not model the future in a vacuum.
```

R2 builds local futures inside an active motivational field: hunger, safety,
sleep, reproduction, social pressure, pain avoidance, curiosity, status, and
other organism-level pressures. R3 checks and repairs models in relation to
that field.

Review question:

```text

```

## Instruction Runtime

The structuration source shifts the practical unit from "idea" to executable
instruction.

Candidate reading:

```text
behavior = winning instruction packet in an activation field
```

An instruction packet includes:

- instruction: what to do;
- trigger: when it activates;
- priority: how strongly it wins against alternatives;
- activation cost: how easily it starts;
- validation state: whether its causal model is still trusted;
- compilation state: declaration, working habit, ritual, superstition,
  discarded rule, or stable skill.

This supports a compact learning cycle:

```text
R3 analyzes or repairs -> instruction is formed -> R1 stabilizes execution
```

## Ritual, Superstition, and Bad Instructions

The batch supports three distinctions worth preserving:

```text
ritual = repeated sequence without sufficient causal reconstruction
superstition = R3-like causal explanation with insufficient causal resolution
bad habit = instruction stabilized into R1 despite poor global fit
```

This lets R Theory diagnose not only whether a system has instructions, but
whether the instruction is valid, activated at the right time, and compiled at
the right level.

## R4 Boundary

The batch strongly argues against treating R4 as a normal next R level.

Candidate rule:

```text
R4 is not accepted as a new operator by default.
```

Apparent R4 effects should first be interpreted as:

- resource limits of R3;
- architecture/scaffolding limits;
- support tooling around R3;
- inability to rewrite every foundation at once.
