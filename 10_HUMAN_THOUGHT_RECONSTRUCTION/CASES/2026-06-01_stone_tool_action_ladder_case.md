## Question

What can the early stone-tool record tell us about changing human thought if we
do not read mental states directly from artifact form?

Test lane:

```text
Oldowan -> Acheulean -> platform preparation -> action grammar
```

## Guardrail

This case is not a ladder of human superiority.

It is a reconstruction of increasingly visible operation structure:

```text
produced edge
-> form-target sequence
-> preparation of future removals
-> grammar-like action organization
```

The case remains non-canon.

## Key Distinction

Oldowan and Acheulean should not be treated as two objects on a shelf.

HTR reads them as different visibility windows into operation:

```text
Oldowan:
  make an edge

Acheulean:
  maintain an evolving form target across removals

platform preparation:
  alter the current surface to make a later operation possible or better

prepared core:
  turn the core into a configured production surface for a desired future flake
```

The central shift is not "more beautiful tool." It is:

```text
from acting on material
to acting on the conditions of future action
```

## Extracted Operator: Future-Operation Preparation

Working formula:

```text
current material state
-> deliberate local modification
-> changed probability/quality of later action
-> more constrained future output
```

This is visible in platform preparation because the operation is not primarily
the final edge or the final tool shape. It is a supporting operation that changes
the action field for subsequent removals.

HTR name:

```text
future-operation preparation
```

Status:

```text
operator_candidate / source_anchored / non-canon
```

Even before writing, a created object can preserve traces of nested action

## What The Case Adds

### 1. The artifact record can preserve sub-operations

Platform preparation is important because it is not the final product. It is a
sub-operation that can be extracted from sequence structure.

This strengthens HTR:

```text
artifact traces can preserve operation architecture, not only output form
```

### 2. Prepared material can become a proto-interface

This does not mean stone tools are "software." It means a prepared core has an
interface-like property in a very limited sense:

```text
the current configuration exposes, blocks, or improves later possible actions
```

That analogy must remain constrained. It is useful only as an operator bridge,
not as a historical equivalence.

### 3. Coexistence breaks the simple progress story

Oldowan and Acheulean co-occur in the Kokiselei record. Therefore the case
should not be read as:

```text
old mode disappears -> new mode replaces it
```

Better:

```text
multiple operation regimes can coexist, specialize, or fail to transmit
```

This is important for the full HTR program: history of thought will likely look
like branching carriers and regimes, not one staircase.

## Failure Pressure

Demote or revise if future work shows:

- platform preparation is too dependent on modern expert replication to map back
  into archaeology;
- Oldowan variability is too wide for "portable action category" without more
  subtyping;
- Acheulean form-target language overstates the intentional uniformity of
  bifaces;
- Levallois/prepared-core material is too late or too separate to belong in
  this same ladder;
- environmental/raw-material constraints explain more than the operation model.

## Provisional Result

The stone-tool lane strengthens HTR in a precise way:

```text
human-created traces can preserve not only products, but action architecture
```

The best recovered operator is:

```text
future-operation preparation
```

This is a strong bridge concept for later comparison with writing, mathematics,
software, and AI interfaces, because each later carrier also changes not only
what humans store, but what future operations become possible.
