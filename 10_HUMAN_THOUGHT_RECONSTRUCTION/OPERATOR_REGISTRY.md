# HTR Operator Registry

This registry records the non-canon operator schema and its instances derived from case studies of the **Human Thought Reconstruction from the Total Artifact Record** research program.

It acts as a coordination boundary to prevent naming drift across case study lanes and to define explicit verification and promotion rules.

---

## 1. Shared Schema: Carrier-Conditioned Operation Shaping

Every registered operator in this module is an instance of the following shared schema:

```text
Modify a present carrier C so that a later operation O becomes
more probable, more constrained, more recoverable, more repeatable, or more
visible than it would be without the modification.
```

The defining cognitive invariant is the redirection of effort from the target action ($O$) onto the *conditions* of the target action ($C$).

---

## 2. Discriminating Axes

Instances are compared and classified along four discriminating axes:

*   **Axis 1 - Temporal gap**: The distance in time between preparing $C$ and executing $O$.
*   **Axis 2 - Agentive locus**: Who executes $O$ relative to who prepared $C$ (self vs. other, human vs. machine).
*   **Axis 3 - Externalized layer**: What part of the operation is offloaded or pushed into the carrier $C$.
*   **Axis 4 - Executability**: Whether the carrier $C$ itself can run $O$, or only condition its execution.

---

## 3. Registered Instances

### I. Future-Operation Preparation (Stone Tool Lane)
*   **Carrier $C$**: prepared core, platform, prepared volume.
*   **Later operation $O$**: physical fracture, shaping strike, core reduction, tool use.
*   **Temporal gap**: short (within the same knapping session).
*   **Agentive locus**: self (the same knapper).
*   **Externalized layer**: material/physical configuration (shaping the core volume to constrain the paths of future strikes).
*   **Executability**: no (the prepared core only conditions the strike; the knapper must execute it).
*   **Reference**: `CASES/2026-06-01_stone_tool_action_ladder_case.md`

### II. Deferred-Operation Surface (Writing / Notation Lane)
*   **Carrier $C$**: token, envelope/bulla, tablet, abstract numeral, phonetic sign.
*   **Later operation $O$**: recall, audit, calculation, cross-checking, institutional authorization.
*   **Temporal gap**: long, open-ended (from days to years; often across generations).
*   **Agentive locus**: other persons, institutions, or administrative offices.
*   **Externalized layer**: the state to be operated on (recording quantities, obligations, and identities).
*   **Executability**: no (the tablet only stores the state; a human clerk must execute the calculation).
*   **Reference**: `CASES/2026-06-01_writing_notation_lane_case.md`

### III. Executable Action Menu / Live Operation Grammar (Software / Interface Lane)
*   **Carrier $C$**: outline, hypertext node, index card, web page, AI conversational workspace.
*   **Later operation $O$**: hyperlink navigation, script execution, prompt submission, tool invocation.
*   **Temporal gap**: near-zero (selected and run in the same interactive cycle).
*   **Agentive locus**: human user and machine carrier jointly.
*   **Externalized layer**: the operation grammar itself (the menu/affordance set of what operations are currently permitted).
*   **Executability**: yes (the carrier itself runs the script, follows the link, or processes the tool-call).
*   **Reference**: `CASES/2026-06-16_software_interface_lane_case.md`

## 4. Promotion and Canon Gates

Nothing in this registry is accepted as theory canon. The registry is a draft candidate mapping.

To propose a schema or any instance for **canon promotion**, the following gates must be met:

1.  **Diagrammatic/Mathematical Lane**: Add a fourth, instrument-independent lane (e.g. geometric proofs or algebraic notations) to test if the four-axis schema survives abstract symbol systems built for proof rather than storage or execution.
2.  **Collapse Test**: Run the collapse test to prove that the schema cannot be fully reduced with no loss to Gibsonian affordance theory, Clark/Chalmers' extended mind thesis, or biological niche construction.
3.  **Self-Reference Resolution**: Source at least one carrier-self-execution case outside the modern AI-workspace lineage to resolve the self-reference boundary of the software/interface lane.
