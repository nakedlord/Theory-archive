## Purpose

This source note supports a concrete textual stress-test of the hypergraph formalization probe on the ending of Mark:

```text
Mark 16:8 and the longer ending tradition, Mark 16:9-20
```

The goal is not to decide the final textual-critical question. The goal is to test whether hypergraph modeling helps separate textual, narrative, theological, institutional, and instructional bundles better than a pairwise graph or simple layer table.

## Verified / Stable Textual Data Used

### 1. Abrupt ending at Mark 16:8

Several early witnesses end Mark at 16:8, including Codex Sinaiticus and Codex Vaticanus.

Use in this test:

```text
Mark 16:8 is treated as a textual rupture / abrupt ending problem.
```

### 2. Longer ending tradition, Mark 16:9-20

The longer ending is present in many later manuscript traditions and is often printed in modern Bibles with brackets or notes marking textual uncertainty.

Use in this test:

```text
Mark 16:9-20 is treated as a later closure / stabilization tradition, not as assumed original Markan text.
```

### 3. Scholarly uncertainty

The test does not need to resolve whether Mark intentionally ended at 16:8, whether an original ending was lost, or whether later endings functioned as reception/stabilization.

Use in this test:

```text
The uncertainty itself is the object: different transmission bundles treat the ending problem differently.
```

## Applicability to Culture as Externalized Instructions

The Mark ending problem is useful because it contains several possible layer-bundles:

```text
textual witness bundle
narrative closure bundle
theological proclamation bundle

instructional transmission bundle
```

If hypergraph modeling is useful, it should separate these without claiming that any one layer is the whole truth.
