## Purpose

This patch moves the instruction-runtime residue out of R core and into Culture
as Externalized Instructions.

It does not update Culture canon.

## Candidate Model

Culture can be read as a storage and transmission layer for instruction packets.

An instruction packet may include:

- instruction: what to do;
- trigger: when it activates;
- priority: how strongly it wins against alternatives;
- activation cost: how easily it starts;
- validation state: whether its causal model is still trusted;
- compilation state: declaration, working habit, ritual, superstition,
  discarded rule, or stable skill.

Working formula:

```text
behavior = winning instruction packet in an activation field
```

## Relation To R Theory

R Theory owns the learning/choice-side bridge:

```text
R3 repairs a rule; R1 stabilizes execution of the repaired instruction.
```

Culture owns the external storage, transmission, reinforcement, and
reactivation of instruction packets across people, artifacts, texts,
institutions, rituals, and environments.

## Candidate Distinctions

Ritual:

```text
repeated sequence with insufficient or no longer reconstructed causal model
```

Superstition:

```text
R3-like causal explanation stabilized without sufficient causal resolution
```

Bad habit:

```text
instruction stabilized into R1 despite poor global fit
```

These distinctions are diagnostic candidates, not canon.

## Boundaries

This patch does not claim:

- every cultural form is reducible to an instruction packet;
- ritual is false or useless by default;
- superstition is merely irrational behavior;
- bad habits are only individual failures;
- instruction runtime is already empirically verified.
