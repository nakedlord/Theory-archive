## Назначение

Модуль `02_SYNTAX` имеет `error_architecture` — классы ошибок **мышления пользователя**. Он не имеет error log **самого SynTax**: где механизм протокола дал худший результат.

Этот файл закрывает пробел. Он фиксирует **фальсификации механизмов, добытые прогонами** — единственный внешний якорь SynTax, оценивающий механизм по результату, а не по логике внутри рамки. Функциональный аналог `01_R_THEORY/ARCHAEOLOGY/` для R-Theory.

## Уроки как эпистемические пробелы (кандидаты в OPEN_GAPS)

1. **Защита вредит генерации.** M1, M4, M6, L2 — все четыре пришлось выключить или ослабить на Design/Расходящихся. Это не частные баги: **защитная архитектура систематически душит порождение.** Отсюда сдвиг v4.0→v5.0 (защитная → субъектная). Обобщение: механизм, полезный на Critique, вероятно вреден на Design. Проверять каждый механизм **по типу задачи**, не глобально.

2. **Симметричные механизмы асимметрично полезны.** M7 требовалось **инвертировать** по типу задачи, а не отключить. Значит некоторые защиты не «включить/выключить», а **развернуть по знаку**.

3. **Sycophancy не ловится авто-механизмом.** M5 провалился на медленном дрейфе. Признано в identity-слое v5.0 как принятое ограничение: *prompt-only защиты снижают частоту, не устраняют.*

4. **Self-correction без внешнего якоря может ухудшать.** Принято в identity v5.0. Это прямо ставит под вопрос M6 и `/ревизия` как самодостаточные.

## Открытое

- Протоколы самих симуляций в архиве отсутствуют (`needs source verification`).
- Ни один механизм v5.0 (гейты, режимы, H5/H7/H10, G2+G6) **не записан в архив как спека** — error log ссылается на конструкции, которых в `02_SYNTAX` нет. Требуется сперва внести runtime v5.0.
- Симуляции прогонялись агентом, инструктированным сравнивать архитектуры — то есть **изнутри рамки**. Это ослабляет их как внешний якорь. Настоящий внешний якорь = задачи, чей исход определяется вне протокола (код прошёл тесты; рекомендация сработала в GYMKiDs; предсказание подтвердилось литературой).
