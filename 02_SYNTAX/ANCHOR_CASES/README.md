## Зачем

`OPEN_GAPS` 11b.1: **у runtime отсутствует внешний якорь.**

`ERROR_LOG.md` содержит симуляционную эмпирику (24/70/72 прогона), но она **внутрирамочна**: прогонялась агентом, инструктированным сравнивать архитектуры SynTax, по критериям SynTax. Это самосогласованность, не валидация.

Эта папка — попытка построить то, чем SynTax не обладает: **оценку по исходу, определённому вне протокола.**

Функциональный аналог: `01_R_THEORY/ARCHAEOLOGY/` — факты, не подчиняющиеся теории.

## Критерий приёма кейса

Кейс принимается, только если выполнены **все три**:

1. **Внешний арбитр.** Исход определяет не оценка качества ответа, а событие вне протокола: тест прошёл/не прошёл; бизнес-метрика сдвинулась/не сдвинулась; предсказание подтвердилось независимым источником/опровергнуто.
2. **Арбитр не читал SynTax.** Литература, тесты, рынок, реальность.
3. **Предсказание сделано ДО проверки.** Ретроспективная подгонка не считается.

Кейс, не проходящий все три, идёт в `ERROR_LOG.md` (внутренняя эмпирика), не сюда.

## Что НЕ является якорем

- «Ответ был глубоким» — оценка внутри рамки.
- «Пользователю понравилось» — оценка внутри рамки.
- Симуляции сравнения архитектур — оценка внутри рамки.
- Progress delta (v3.0) — меряет дельту понимания, не соответствие миру.

## Известное ограничение всех LLM-кейсов

Агент мог **знать** подтверждающий источник из обучающих данных и «вывести» его задним числом, не осознавая. Отличить реальное предсказание от скрытого припоминания изнутри **невозможно**.

Смягчение: предпочитать кейсы, где арбитр появился **после** cutoff агента, либо где исход количественный и не мог быть угадан (бизнес-метрика, результат теста).

Это ограничение фиксируется в каждом кейсе, а не замалчивается.
