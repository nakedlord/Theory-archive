## Purpose

This note routes Q-R3 out of R core canon and into the SynTax operator side.

It does not promote Q-R3 into SynTax canon or the formal operator registry.

## Candidate Function

Q-R3 preserves productive problem tension before premature closure.

Working loop:

```text
detect unresolved frame -> hold answer suppression -> generate better question
-> test frame shift -> release answer only when the frame improves
```

The operator is useful when an answer is available too early but would freeze a
weak frame.

## Candidate Moves

- REFINE: sharpen the current question without changing the frame.
- SHIFT: move to a neighboring frame that explains the mismatch better.
- META-SHIFT: question the rule that selected the current frame.

## SynTax Fit

Q-R3 is closer to SynTax than to R core canon because it is a procedure for
managing reasoning state:

- preserves unresolved tension;
- controls when answers are allowed;
- generates questions;
- tests whether a frame improves;
- prevents local closure from pretending to be global repair.

## Failure Modes

- infinite questioning after enough evidence is available;
- performative complexity without real frame improvement;
- answer suppression used as avoidance;
- self-confirming questions that protect the current frame;
- drift into R4-like rhetoric instead of concrete R3 repair.
