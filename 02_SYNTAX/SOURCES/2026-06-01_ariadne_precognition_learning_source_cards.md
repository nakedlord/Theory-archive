## Purpose

These cards support a bounded SynTax/Ariadne mechanism:

```text
theory or research signal
-> precognitive source probe
-> operator extraction
-> self-application hypothesis
-> safe trial
-> outcome evaluation
-> reviewed promotion / demotion
```

The target is not model fine-tuning or hidden autonomy. The target is a
reviewable learning loop where Ariadne improves its own operator surface,
routes, prompts, proof gates, playbook recipes, and CSP-facing briefs through
source-backed candidate patches.

- `02_SYNTAX/HYPOTHESES/2026-05-23_ai_assisted_precognition.md`
- `02_SYNTAX/SOURCES/2026-05-23_ai_assisted_precognition_source_note.md`

- the distinction between summary and precognition;
- the formula `source signal -> AI probe -> value map -> selective attention
  -> transfer/application`;
- the idea that AI can act as a scout before the user spends deep attention.

```text
source is first probed for what it can do for the user's system
```

Does not support:

- skipping primary-source checks;
- automatic canon or system mutation.

## Source Card 3: Reflexion

Citation:

- Noah Shinn et al., "Reflexion: Language Agents with Verbal Reinforcement
  Learning", arXiv, 2023.
- URL: https://arxiv.org/abs/2303.11366

Used for:

- the idea that language agents can improve behavior by generating verbal
  reflections after feedback and storing those reflections for later attempts.

Supports:

```text
experience -> feedback -> verbal reflection -> future behavior change
```

Boundary for Ariadne:

```text
reflection becomes a reviewed learning packet, not hidden memory authority
```

## Source Card 4: Self-Refine

Citation:

- Aman Madaan et al., "Self-Refine: Iterative Refinement with Self-Feedback",
  arXiv, 2023.
- URL: https://arxiv.org/abs/2303.17651

Used for:

- draft -> feedback -> refinement as a general loop;
- separating generation from critique before accepting output.

Supports:

```text
candidate patch -> critique -> revised candidate patch
```

Boundary for Ariadne:

```text
refinement improves a proposal; it does not apply it without a gate
```

- Shunyu Yao et al., "ReAct: Synergizing Reasoning and Acting in Language
  Models", arXiv, 2022.

- treating tool use as an evidence-gathering step rather than a black-box
  answer jump.

Supports:

```text
reason -> bounded action/read -> observation -> revised reasoning
```

Boundary for Ariadne:

```text
read/action steps remain inside CSP tool classes and permission gates
```

## Source Card 9: Double-Loop Learning

Citation:

- Chris Argyris and Donald Schon, "Organizational Learning: A Theory of Action
  Perspective", 1978.

Used for:

- distinguishing single-loop correction from double-loop revision of governing
  assumptions.

Supports:

```text
single-loop: improve an action
double-loop: improve the rule that selected the action
```

Boundary for Ariadne:

```text
double-loop changes must be proposed as explicit operator/proof-gate patches
```

## Combined Boundary

Allowed:

- use AI-assisted precognition to probe sources for system relevance;
- extract operators and self-application hypotheses;
- store reviewed learning packets as candidate patches;
- test patches in read-only or bounded producer mode;
- promote only after evidence, human/consolidator review, and repo guardrails.

Not allowed:

- hidden self-modification;
- silent canon promotion;
- unsupervised connector, local filesystem, product repo, or CSP projection
  mutation;
- using retrieved memory as authority over source files;
- treating a source summary as proof that a system change works.
