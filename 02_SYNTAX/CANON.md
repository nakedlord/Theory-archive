# SynTax Architecture

Текущий канон: v5.3 / two-layer: governance infrastructure + reflexive runtime theory.

## Короткое определение

SynTax — архитектура ведения мышления, целей, канона, патчей, ошибок, контекстных модулей и эволюции теорий.

SynTax состоит из двух слоёв.

**Governance-слой** (патчи v1.9..v5.2) — инфраструктура: удерживает направление, различает канон и гипотезы, управляет ошибками, не смешивает рабочие мутации с признанными конструкциями. Не является содержательной теорией мира.

**Runtime-слой** (`RUNTIME_SPEC.md` v5.0) — **рефлексивная теория мышления**: содержит генеративное ядро, производящее содержание. Её объект (мышление агента) изменяется от факта применения теории.

Главное различение: инфраструктура не порождает — она хранит и маршрутизирует. Генеративное ядро порождает, следовательно runtime содержателен.

Ограничение: SynTax — теория *мышления*, не теория *мира*. Механизмы runtime фальсифицируемы поштучно (Gᴛ); ядро оценивается по порождённому мышлению (Gᴀ, патч v5.2). Внешний якорь runtime отсутствует — симуляционная эмпирика внутрирамочна (см. `ERROR_LOG.md`).

Патч: `PATCHES/v5.3_reflexive_theory_status.md`.

## Слои governance-инфраструктуры

- Goal Layer — память направления и целей.
- Memory Infrastructure — канон, карта, глоссарий, версии, зафиксированные узлы.
- Evolution Infrastructure — гипотезы, мутации, патчи, отбор, kill-механизмы.
- Context Arbitration — выбор подходящей рамки чтения без скрытого lock-in.
- Error Architecture — классы ошибок, повторяемость ошибок, drift detection.
- Operator Registry — управляемый каталог операторов мышления.

## Базовое правило

Канон и лаборатория гипотез не должны смешиваться. Новая идея сначала проходит через гипотезу, кейсы, проверку связности и только затем становится патчем канона.

## Runtime-слой

`RUNTIME_SPEC.md` — активная спека исполнения: четыре входных гейта, режимы сессии, фильтры L1-L6, механизмы M1-M7, генеративное ядро G2+G6, условные модули H5/H7/H10.

`ERROR_LOG.md` — фальсификации механизмов, добытые прогонами (M3 удалён, M5 разжалован, M7 инвертирован, M1/M4/M6/L2 душат генерацию на Design).

## Статус

SynTax используется как операционная система архива (governance) и как рефлексивный scaffold мышления агента (runtime).
