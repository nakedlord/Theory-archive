## Purpose

This plan separates Boundary Saturation's internal model language from external
physics evidence. The current module remains a speculative formal model. The
M5 chat export is provenance for the idea's development, not a physics source.

## Verification Rule

Each future source pass should record:

```text
external source -> factual claim -> Boundary reading -> limitation -> status
```

No claim should move from analogy or candidate motif into canon without an
external source, a limitation note, and an explicit status update.

## Status Categories

- `supported`: directly supported by a source and compatible with the module.
- `plausible`: compatible but not directly established.
- `contested`: disputed, interpretive, or source-dependent.
- `speculative`: internal model language without external support.
- `rejected`: contradicted or too misleading to keep.
- `needs_source`: not yet checked.
